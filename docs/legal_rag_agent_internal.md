# 法律 RAG 知识库助手项目记录（给自己看的版本）

## 1. 这个项目本质上做了什么

这不是“训练了一个法律大模型”，而是做了一套面向法律问答的 RAG 知识库助手系统。核心思路是：

1. 本地知识库负责存法律文本和外部数据库内容。
2. 本地检索层负责先把候选证据找出来。
3. 大模型不直接查全库，而是负责改写问题、判断相关性、组织答案和做校验。
4. 回答必须尽量和证据绑定，并且给出引用。

所以对外讲的时候，重点不要放在“模型创新”，而是放在“法律问答场景下的 RAG 架构创新和工程创新”。

## 2. 已实现的核心功能

### 2.1 数据入库

已经支持：

1. PDF、DOCX
2. CSV、JSONL
3. SQLite 外部数据库文件

数据会统一进入外部持久化库，形成：

1. SQLite 文档库
2. FAISS 向量索引
3. TF-IDF 稀疏索引

### 2.2 检索模式

现在是两种模式：

1. 大模型检索  
   本地先宽召回，大模型从候选里挑相关证据，再回答。

2. 混合检索（大模型 + 小模型）  
   本地检索模型先把范围缩小，再由大模型筛证据和生成答案。

### 2.3 多轮记忆

现在的记忆不是普通聊天上下文拼接，而是分层：

1. 最近 3 轮直接进 prompt
2. 更早历史转成会话级记忆 RAG

这个点比较值得讲，因为它既保留了上下文承接，又控制了窗口成本。

### 2.4 引用与界面

已经有：

1. 流式输出
2. 历史记录
3. 会话管理
4. PDF 引用预览
5. 一键启动 bat

也就是说这个项目已经不是脚本级 Demo，而是一个可交互的本地 Agent 应用。

## 3. 现在真正能说的创新点

### 3.1 检索和生成解耦

这是最稳的点。  
不要说“我让大模型直接做法律推理”，而要说：

“我把检索和生成分开，本地知识库负责查，大模型负责理解和表达，从而降低 token 成本并提升可控性。”

### 3.2 检索前法律语义改写

这个点很适合讲。  
很多用户会问“这算不算”“会不会有责任”“这种情况怎么办”，这种表达和法条语言不一致。现在项目会先把问题转成法律检索表达，比如：

- “那这种情况会不会算防卫过当，还要负刑事责任吗？”

会被改成更接近：

- “第三人制止正在进行的强奸行为致不法侵害人重伤是否构成防卫过当及是否承担刑事责任”

这个中间层很关键。

### 3.3 法域路由 + 多争点拆解

这是现在最像“法律思维”的一层。

复杂问题往往同时涉及：

1. 刑事责任
2. 民事责任
3. 行政责任
4. 程序问题

现在系统会先做法域路由，再拆成多个争点去检索，这比普通 RAG 直接整句搜要更合理。

### 3.4 分层记忆 RAG

这个比普通聊天记忆更高级。  
你可以讲成：

“系统把短期记忆和长期记忆拆开建模，最近对话直接保留语义连续性，更早历史通过会话级 RAG 方式召回，兼顾了连贯性和成本控制。”

### 3.5 证据治理链路

现在不是“检索后直接回答”，而是多了一条证据治理链：

1. 候选召回
2. 相关性筛选
3. 回答生成
4. 回答后自检
5. 引用一致性校验

这个点对法律场景非常重要，因为法律问答最怕“说得像，但证据不真支撑”。

### 3.6 可验证引用

这个点属于产品可信性创新，不是模型创新。  
系统不是简单列来源名，而是尽量把引用落到 PDF 页码和原文片段上，方便用户核查。

## 4. 面试时哪些话能说，哪些不要说

### 可以说

1. 这是本地化法律 RAG 知识库助手，不是单纯聊天机器人。
2. 我做了检索前改写、法域路由、多争点拆解、分层记忆、证据筛选、自检和引用校验。
3. 我的重点是提高法律问答的准确性、可解释性和 token 效率。

### 不建议直接说

1. “我做了一个法律大模型”
2. “我训练了法律模型”
3. “我的系统能直接替代法律专业判断”

更安全的说法应该是：

“这是一个面向法律文本问答的 RAG 知识库助手系统，重点解决法律检索准确性和回答可信性问题。”

## 5. 这个项目当前最强的卖点

如果只能挑 3 个点讲，优先级建议是：

1. 本地检索和大模型生成解耦，节省 token
2. 法律语义改写 + 法域路由 + 多争点拆解，提升复杂法律问题检索准确性
3. 分层记忆 + 引用一致性校验，提升多轮问答可信性

## 6. 当前还存在的边界

这个项目现在已经能讲，但不要讲过头。当前边界包括：

1. 仍然依赖外部大模型做改写、筛选、自检
2. 不是法律专业裁判系统，仍是问答辅助系统
3. 检索质量依然受本地模型、文本切分和候选召回影响
4. 引用一致性虽然增强了，但不是严格符号化证明

## 7. 一句话自我总结

这项目最准确的定位不是“做了法律模型”，而是：

“围绕法律问答场景，搭建了一套本地化、低成本、可验证的 RAG 知识库助手系统，并通过法律语义改写、法域路由、多争点拆解、分层记忆和证据校验等机制，提升法律检索与回答质量。”
