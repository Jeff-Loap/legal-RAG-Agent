# 法律 RAG 知识库助手项目说明（正式版）

## 一、项目概述

本项目面向法律法规问答场景，设计并实现了一套本地化法律 RAG 知识库助手。系统以法律文本和外部数据库为知识底座，结合本地检索模型与大语言模型，完成法规检索、证据筛选、答案组织、引用展示和多轮交互等能力，兼顾回答质量、成本控制、可解释性与工程落地性。

项目目标主要包括以下几个方面：

1. 构建面向法律文本的本地知识库与外部持久化数据库。
2. 将检索与生成解耦，避免大模型直接对全量数据库进行高成本检索。
3. 提升法律问答场景下的检索准确率、答案可信度与引用可验证性。
4. 支持多轮会话、历史管理、流式输出和交互式界面，形成可直接使用的法律问答 Agent。

## 二、项目功能实现

### 1. 多源法律知识入库

系统支持从本地目录递归加载多种数据源，包括 PDF、DOCX、CSV、JSONL、SQLite 等文件，并将新增文档统一纳入外部知识库。入库流程包括文本抽取、结构化分块、元数据生成、向量索引构建和稀疏索引构建，最终落地到 SQLite、FAISS 与 TF-IDF 持久化索引中。

### 2. 本地检索与生成解耦

项目采用“检索层”和“生成层”分离的架构设计。本地检索模块负责从法律知识库中完成候选证据召回，大模型负责对候选证据进行语义改写、证据筛选、答案组织和自然语言生成。该设计有效降低了大模型直接查询数据库所带来的 token 消耗与不可控性。

### 3. 双模式检索能力

系统支持以下两种问答模式：

1. 大模型检索  
   先由本地知识库完成宽召回，再由大模型从候选法条中筛选有效证据并生成回答。

2. 混合检索（大模型 + 小模型）  
   由本地检索模型先完成候选收缩，再交由大模型进行证据筛选、答案整理和最终输出。

该双模式设计使系统能够根据场景需求在成本、速度和准确率之间进行灵活取舍。

### 4. 多轮上下文记忆

项目实现了分层上下文记忆机制：

1. 最近 3 轮对话直接作为原始上下文送入回答链路。
2. 更早历史会话构造成会话级记忆 RAG，并在后续追问时按相关性进行召回。

该设计兼顾了短程上下文承接能力与长程历史记忆能力，避免了普通多轮对话中上下文无限膨胀的问题。

### 5. 法律问答专用提示词链路

围绕法律问答场景，系统构建了分阶段提示词链路，覆盖：

1. 检索前法律语义改写
2. 法域路由判断
3. 多争点拆解
4. 候选证据相关性筛选
5. 分类型答案生成
6. 回答后自检
7. 引用一致性校验

该链路使系统不再是单一 prompt 的问答工具，而是形成了面向法律检索与法律回答的多阶段 Agent 工作流。

### 6. 证据引用与可信性增强

系统在回答中提供结构化引用信息，包括来源文件、页码、片段预览等，并支持在界面中查看 PDF 对应页内容。回答中的引用并非简单附加，而是经过证据筛选和一致性校验后输出，从而增强了答案的可核验性与可信度。

### 7. 交互界面与历史管理

项目基于 Streamlit 构建本地交互界面，支持：

1. 流式回答输出
2. 历史问答记录保存
3. 会话查看与管理
4. 新建会话
5. 引用证据预览
6. 一键启动脚本

该界面能够支持法律问答场景下的连续交互和证据核查，具备较好的使用体验与演示效果。

## 三、项目创新点

### 1. 本地检索与大模型生成解耦

项目将“数据库检索”与“自然语言生成”进行明确拆分，避免大模型直接对外部数据库进行高成本查询。该架构在保证回答质量的同时，显著降低了 token 消耗，提高了系统可控性和工程可落地性。

### 2. 面向法律场景的检索前语义改写

系统在检索前增加了法律语义改写中间层，将用户自然语言问题转换为更接近法律条文表达的检索语句，从而缩小“口语提问”与“法条用语”之间的表达差异，提高了法律场景下的召回效果。

### 3. 法域路由与多争点拆解

针对法律问题往往包含多个责任维度和多个法域的特点，系统先通过大模型识别问题所属法域，再将复杂问题拆解为多个法律争点分别检索。该方式更贴近法律分析流程，相比普通单轮 RAG 具有更强的复杂问题处理能力。

### 4. 分层记忆 RAG

系统并未采用单纯拼接历史对话的传统方式，而是设计了“最近上下文直传 + 更早历史 RAG 召回”的分层记忆机制。该方案既保留了多轮追问所需的上下文连续性，又控制了上下文窗口成本，适用于长会话法律咨询场景。

### 5. 证据级筛选、自检与引用一致性校验

项目在候选证据召回之后，引入了证据相关性筛选、回答后自检和引用一致性校验三个阶段，使系统不仅能够“找到相关内容”，还能够进一步判断哪些条文真正支撑最终结论，从而显著提升回答的证据约束能力。

### 6. 面向可信性的可视化引用设计

系统支持在回答中展示结构化引用，并关联 PDF 页码和证据片段预览，使用户能够从答案直接回溯到原始法条文本。该设计强化了系统的可解释性、可审计性与可信性。

## 四、项目价值

本项目不是单纯的问答 Demo，而是一套具有较强工程落地能力的法律 RAG 知识库助手原型系统。其价值主要体现在：

1. 降低法律智能问答场景中的大模型调用成本。
2. 提升法律知识检索和答案生成的可信性与可验证性。
3. 为后续扩展到更多法律文本、法规库和专业知识库提供稳定架构基础。
4. 具备较好的展示性、可复用性与进一步产品化潜力。

## 五、总结

本项目围绕“法律问答准确性、证据可信性、上下文记忆能力和工程成本控制”四个核心目标，构建了一套本地化法律 RAG 知识库助手。通过本地检索、大模型语义改写、法域路由、争点拆解、证据筛选、自检校验和可视化引用等设计，项目在法律智能问答场景下形成了较完整的功能闭环与较突出的系统创新性。
